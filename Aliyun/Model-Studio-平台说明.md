# 阿里云 Model Studio（百炼）平台说明

> 控制台地址：https://modelstudio.console.alibabacloud.com/

---

## 一、平台是什么

**阿里云百炼（Model Studio）** 是阿里云的一站式**大模型开发与应用构建平台**。不论开发者还是业务人员，都可以通过控制台或 API 使用千问（Qwen）全系列及主流第三方大模型，**无需自建 GPU 与运维**。

- **产品定位**：大模型服务平台（推理 + 应用开发 + 模型调优等）
- **与 DashScope 的关系**：你当前用的 **DashScope API**（如 `dashscope-intl.aliyuncs.com`）就是百炼提供的**模型推理服务**之一；Model Studio 控制台是同一套服务的**管理入口**（创建 Key、看用量、做应用、调优等）。

## 三、核心能力概览

### 1. 模型服务（你已在用的部分）

- **通义千问系列**：Qwen3-Max、Qwen3.5-Plus、Qwen3.5-Flash、Qwen-Coder 等，支持文本、多轮对话、代码、推理等。
- **多模态**：视觉理解（Qwen-VL）、图像生成（Qwen Image、万相）、视频生成（Wan 系列）、语音识别/合成、向量（Embedding）等。
- **第三方模型**：如 DeepSeek、Kimi 等（以控制台/文档为准，部分仅特定地域）。
- **接入方式**：
  - 千问官方 API / **OpenAI 兼容 API**（你用的就是这种：改 `api_key`、`base_url`、`model` 即可）。
  - DashScope SDK。

你在 `Aliyun.py` 里写的 `base_url="https://dashscope-intl.aliyuncs.com/compatible-mode/v1"` + `model="qwen3.5-plus"`，就是**在百炼平台上调用模型**的典型方式。

### 2. 应用开发（低代码/无代码）

在控制台里可以：

- **Prompt 工程**：写提示词、模板，在线调试效果。
- **智能体（Agent）**：配置工具调用、多轮对话、简单记忆。
- **工作流**：把多个模型或步骤串成流程。
- **知识库（RAG）**：上传文档，做“检索 + 大模型”的问答应用。
- **插件**：扩展能力（搜索、代码执行等）。

适合：快速做 Demo、内部工具、和算法/产品协作时在“同一平台”上试效果。

### 3. 模型调优（微调）

- 支持**全参数微调**、**高效微调（如 LoRA 等）**，在平台上做数据准备、训练、评测。
- 与「和算法团队协作、评估客户微调可行性」直接相关：可以在百炼上做简单微调实验，再决定是否上 PAI 等更重型的训练。

### 4. 模型部署与运维

- **部署**：将模型或应用部署为可调用的服务。
- **用量统计与监控**：在控制台看调用量、Token 消耗、成功率、延迟等（模型监控/统计页面）。

## 四、计费与免费额度

- **开通**：开通百炼/Model Studio 本身不收费，**按实际调用计费**（按 Token 或按次，视模型而定）。
- **新用户免费额度**（以当前政策为准，具体见控制台）：
  - 国际站（新加坡）：新用户常有一定**免费 Token 额度**（如百万级），在有效期内使用；用完后按量付费。
  - 可开启「**免费额度用完即停**」，避免自动转为扣费。
- **查看用量与账单**：
  - 控制台 → **模型监控**（按地域进入）→ 看某模型的调用量、Token、成功率等。
  - 阿里云 **用户中心 → 账单/成本分析** 看费用。

---

## 五、安全与合规

- **数据**：你的输入/输出不会用于训练模型；传输与存储有加密。
- **网络**：支持在 VPC、PrivateLink 等隔离环境中使用，适合企业专有数据。
- **内容**：平台侧有内容安全与合规策略，可配合人工干预。

## 六、和你当前项目的关系（实操对应）

| 你做的事 | 在 Model Studio 里对应什么 |
|----------|----------------------------|
| 用 `DASHSCOPE_API_KEY` 调 Qwen | 在控制台 **API-KEY** 页面创建/管理 Key；**模型服务**即你调用的推理能力。 |
| `base_url` 选新加坡 | 选的是 **国际（新加坡）** 地域的推理端点。 |
| 想查调用次数/费用 | 控制台 → **模型监控**（同地域）→ 选模型看统计；**用户中心**看账单。 |
| 想试其他模型或做应用 | 控制台 → **模型体验/模型广场** 试模型；**应用开发** 做 Prompt/智能体/知识库。 |
| 想评估微调 | 控制台/文档中的 **模型调优**；也可结合 PAI、Hugging Face 做更重型的实验。 |

总结：**Model Studio 控制台 = 百炼平台的“总控”**；你写的 **云服务搭建 + AI 模型调用**（`Aliyun.py`）就是在用百炼提供的**模型服务**。要“讲清楚这个平台”，就可以按：**是什么（一站式大模型平台）→ 控制台与地域 → 模型/应用/调优/部署/计费 → 和你已有脚本的对应关系** 来组织。
