# 阿里云 GenAI SA 三面准备（业务专业负责人）

## 一、面试官视角：他会重点看什么

- **你是不是真做过 SA 的活**：需求理解 → 方案设计 → Demo/POC → 落地，而不是只懂概念。
- **对阿里云 AI 产品是否认真了解**：百炼、Qwen、Wan，以及和竞品的差异。
- **能否独立扛一个客户项目**：从售前到方案到 Demo 到内部协同。
- **业务深度**：某个行业/场景能讲清楚「为什么用大模型、怎么用、架构长什么样」。

## 三、建议重点准备的 3～5 个问题（按优先级）

### 1. 你做过的最能体现 SA 价值的项目/案例？（必问）

**答题结构：**  
背景（客户/业务） → 需求（业务诉求 + 技术诉求） → 你的角色（需求分析、方案设计、Demo/文档） → 方案要点（模型/ RAG/Agent/Prompt 等） → 结果（落地情况、客户/销售反馈） → 复盘（哪里做得好、哪里会改进）。

**注意：** 尽量选和「大模型 / RAG / Agent / 云」相关的经历；如果没有直接经历，用学习项目/内部项目，但要诚实说明角色边界。

---

### 2. 你对阿里云百炼、Qwen、Wan 了解多少？和 OpenAI/Claude 比怎么选型？

**建议提前做：**  
- 打开阿里云百炼官网/文档，看：百炼是什么（MaaS 平台）、Qwen 与 Wan 的定位（通用 vs 多模态/场景）、计费方式、主要 API 形态。  
- 准备一句对比：例如「Qwen 对标通用对话与长文本，Wan 更偏多模态/特定场景；和 OpenAI 比，我们会强调成本、数据合规、定制化空间」。

**话术要点：** 如实说「目前对百炼的实操还在学习，但用过 OpenAI/Claude/其他国产模型，对 MaaS 选型逻辑是共通的」，然后接你的选型思路（场景、时延、成本、合规）。

---

### 3. 某个具体场景的解决方案设计（RAG / Agent / 客服 / 知识库 等）

**示例问题：** “客户要做企业知识库问答，你会怎么设计整体方案？”

**答题框架：**  
- **场景与目标**：谁用、解决什么问题、成功标准（准确率、时延、成本）。  
- **数据**：数据从哪来、格式、量级、更新频率。  
- **架构**：数据接入 → 清洗/分块 → 向量化与检索（RAG）→ 大模型调用 → Prompt 设计 → 前端/API。  
- **模型与产品**：为什么用 RAG 而不是纯 fine-tune；为什么选某类模型（长文本、多语言等）。  
- **风险与优化**：幻觉、检索不准、时延；你怎么通过 Prompt、检索策略、评估迭代来优化。

准备 1 个你熟悉的场景，按上述结构能讲 3～5 分钟。

---

### 4. 和销售、产品、算法协作的经验

**可能问法：** “SA 和销售怎么分工？”“客户提了一个产品没有的功能，你会怎么处理？”

**要点：**  
- 销售：负责商机、关系、合同；SA 负责需求澄清、方案、Demo、标书技术部分；你如何从销售给的信息里提炼技术需求。  
- 产品：客户需求先判断是「项目定制」还是「产品能力缺失」；若是后者，归纳成清晰的需求（场景+价值+建议），再走内部流程。  
- 算法：微调、模型选型、效果优化时，你提供场景和数据理解，算法做模型与实验；能说清楚「我会怎么配合」即可。

---

### 5. 为什么想做 GenAI SA、为什么选阿里云？

**建议话术逻辑：**  
- 一二面已经确认你更适合 SA：你喜欢把「技术」和「业务」连起来，享受做方案和 Demo、推动落地，而不是纯商务拓展。  
-  GenAI 正在改变各行业，SA 是离客户和场景最近的技术角色，你想在这个窗口期深入做下去。  
- 阿里云有国内领先的大模型与百炼生态、海外业务也在拓展，你想在「大模型 + 云 + 国际化」这条线上积累。

## 四、今天可以立刻做的事（优先级排序）

1. **查阿里云百炼 + Qwen/Wan**  
   - 官网/文档各看 15～30 分钟，记下：产品定位、主要能力、和竞品差异、计费大致逻辑。  
   - 若时间紧，至少能说出「百炼是阿里云 AI MaaS 平台，Qwen/Wan 是上面的核心模型，支持 API、RAG、Agent 等能力」。

2. **写一个「SA 价值项目」逐字稿**  
   - 按上面「背景→需求→角色→方案→结果→复盘」写 1 页，讲熟，控制在 3 分钟内。

3. **准备一个 RAG 或 Agent 方案口述**  
   - 选一个场景（知识库/客服/内部工具都行），用「场景→数据→架构→模型→风险」过一遍，能讲 2～3 分钟。

4. **想好「与销售/产品协作」的一个具体例子**  
   - 哪怕是从需求到方案到内部沟通的小故事，避免只说「我会配合」。

5. **微调加分项**  
   - 若时间允许：搜一下 LoRA、QLoRA、PEFT 的简短介绍，能说「我会和算法同事一起看数据量和成本，再决定是否微调、用哪种方式」。

## 五、面试中的小技巧

- **先听清再答**：业务负责人问题可能带行业术语，没听清就说“您刚才是说 xxx 吗”，确认后再答。  
- **用 STAR 但别背稿**： Situation/Task 简短，Action 和 Result 多讲一点，尤其是你的判断、取舍、迭代。  
- **不会的诚实说**：例如「百炼我目前更多是看文档，还没在生产环境用过，但我会用类似平台做 RAG/Agent，逻辑是相通的」，再转到你熟悉的部分。  
- **收尾可反问**：例如「团队目前 GenAI 项目里，SA 和销售、产品的协作节奏是怎样的？」「这个岗位会主要 support 哪几个行业或区域？」显得你关心实际工作方式。

---

## 六、一句话总结

三面要证明三件事：**你真做过 SA 的活（有案例）、你对阿里云 AI 产品有诚意（做过功课）、你能在业务负责人面前把「需求→方案→落地」讲清楚（有框架、有细节）。** 把上面 3～5 个问题准备好，再补上百炼/Qwen/Wan 的功课，就可以比较稳地上场。

祝你三面顺利。
