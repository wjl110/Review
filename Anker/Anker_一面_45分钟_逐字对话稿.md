## 第一题：RAG 项目深挖（2:00-8:00）

**候选人**：这个项目背景是微博内部知识分散在 Wiki、各种文档，员工检索效率低。我的目标是用自然语言问答帮大家快速获取准确答案。

切块策略上，我一开始试了固定长度切块，但发现会把段落切断。后来改成**按语义边界切块**：段落级为主，遇到代码块或表格特殊处理，还加了 20% 的重叠窗口防止上下文割裂。

Embedding 模型我们测试了 BGE-large、M3E、Ada-002，最终选 **BGE-large-zh**，中文场景性价比最优。向量数据库用的 Milvus，亿级向量检索延迟能控制在 100ms 内。

**面试官**：幻觉控制怎么做的？这个应该是 RAG 的大难题。

**候选人**：确实，我们早期幻觉问题很严重。我的解法是三层：

第一层是**检索层**，Top-K 结果相关性得分必须大于 0.75 才进 LLM 上下文；第二层是**生成层**，强制要求 LLM 标注信息来源，不确定时触发 "需要更多信息" 而非瞎编；第三层是**反馈层**，收集用户 thumbs up/down，Bad Case 每周复盘。

这样把 thumbs down 率从早期的 40% 降到了 12%。

**面试官**：如果要把这套 RAG 迁移到 eufyMake 的创作知识库，比如 3D 打印故障排查、材料参数，你会做哪些调整？

**候选人**：这是个好问题。我觉得核心差异是**知识形态**——

企业内部 RAG 主要是文档型知识，但 3D 打印需要大量结构化数据，比如材料参数表、故障码对应关系。我会引入 **SQL + 向量混合检索**：参数类查询走结构化查询，经验类查询走语义检索。

另外 eufyMake 场景可以扩展**多模态能力**：用户上传打印失败的照片，视觉检索匹配相似故障案例。这对新手特别有价值。

**面试官**：RAG 和 Fine-tuning 怎么选？如果我们要做一个 AI 设计助手，你会怎么选？

**候选人**：我的判断是**混合方案**——

RAG 适合知识更新频繁、需要溯源的场景，比如材料库、故障库；Fine-tuning 适合需要特定风格输出的场景，比如设计建议的语气、格式。

eufyMake 的 AI 设计助手可以用 RAG 做知识底座，Fine-tuning 优化输出风格。如果初期数据量不够，可以先纯 RAG，等积累了足够对话数据后再加 Fine-tuning。

## 第二题：AI 创作工具深度（8:00-15:00）

**面试官**：Boss 上你提到用过 Lovart、即梦、可灵。以 Lovart 为例，你觉得它相比传统设计工具的核心差异是什么？

**候选人**：我用 Lovart 做个人产品的视觉设计，最大的感受是它**改变了设计的工作流**——

传统是 "先学软件再创作"，Lovart 是 "先描述想法再微调"。输入方式从操作画布变成自然语言，迭代模式从手动绘制变成选择-生成-微调。

但局限性也很明显：对精准的品牌规范控制还不够，比如 LOGO 位置、字体层级的精细调整还是得用传统工具。

**面试官**：如果让你设计 eufyMake 的 AI 设计功能，你会借鉴 Lovart 的哪些点？

**候选人**：我会借鉴三个点：

第一，**多版本并行生成**，一次 Prompt 出 3-4 个方案，降低用户选择焦虑；第二，**视觉反馈闭环**，生成后支持区域重绘、风格迁移；第三，**模板工作流**，从描述直接生成多场景物料。

针对 eufyMake 的 3D 打印场景，我设想的是 **Vibe Design 接入**：用户描述 "赛博朋克风格的手机支架"，AI 生成 3D 模型预览 + 打印参数建议 + 可制造性检查。把 "设计 → 验证 → 打印" 的链条缩短到几分钟。

**面试官**：3D 打印场景下，AI 生成模型面临什么独特挑战？

**候选人**：我想到三个核心挑战——

**水密性**：AI 生成的 mesh 常有破面，打印会失败，需要后处理修复；**支撑结构**：复杂结构需要可打印性检查，AI 应给出支撑建议；**耗材匹配**：PLA 和 ABS 对模型结构要求不同，AI 生成时要考虑材料因素。

我的解决思路是**生成后接切片软件 API 做预检**，不通过就反馈给用户调整，而不是让用户打完才发现失败。

**面试官**：你用过 ComfyUI，如果要在 eufyMake 里落地一个类似的工作流编辑器，你会怎么设计？

**候选人**：我会做**用户分层**：新手用预设模板，进阶用户用节点编辑。

节点设计抽象为「输入 → 处理 → 输出」三层，隐藏底层参数。比如「文生 3D 模型」节点，内部封装了 prompt 解析、模型调用、mesh 修复，用户看不到具体参数。

错误反馈很重要，节点连线错误时给出可视化提示，而非报错堆栈。还要支持工作流社区共享，类似 ComfyUI 的 JSON 导出机制。

## 第三题：用户洞察与创作者分层（15:00-22:00）

**面试官**：eufyMake 的用户是 DIY 创作者，从 Hobbyist 到 Professional 跨度很大。你在微博做 AIGC 时是怎么做用户分层的？

**候选人**：我在微博用了三层分层法——

**按使用频次**：尝鲜型（1-2次）vs 工具型（每周）vs 专业型（每日）；**按内容类型**：社交图文 vs 营销物料 vs 影视级制作；**按协作方式**：个人创作 vs 团队协作 vs MCN 批量生产。

洞察方法是埋点数据 + 深度访谈 + 社群反馈三位一体。

**面试官**：如果让你来做 eufyMake 的用户洞察，你会怎么设计调研方案？

**候选人**：我会分四层用户设计——

**入门爱好者**：低门槛、教程丰富、模型库；**进阶 Maker**：参数调节、多材料支持、社区；**专业设计师**：专业工具链、批量生产、API；**工作室/企业**：品牌定制、批量管理、售后。

调研方案是定量问卷 + 定性深访 + 可用性测试 + 社群潜水。特别要关注 Reddit r/3Dprinting、拓竹社区，收集原生需求。

**面试官**：3D 打印的核心痛点是打印失败率高，从产品经理角度，你会怎么降低失败率？

**候选人**：四个层面——

**前置检查**：AI 模型检测悬垂角度、壁厚，预警可能失败；**智能支撑**：自动生成最优支撑，易拆除且不影响表面；**实时监控**：摄像头 + 视觉识别，发现翘边及时暂停；**社区知识库**：同类机型 + 材料的成功参数一键复用。

关键是把 "失败后排查" 变成 "打印前预防"。

## 第四题：失败与迭代（22:00-28:00）

**面试官**：你简历提到收集 58 条反馈驱动迭代。能讲一个具体的失败案例吗？

**候选人**：我讲 RAG 智能体早期的幻觉问题。

上线第一周，thumbs down 率高达 40%，用户反馈 "AI 胡编乱造"。典型案例是问 "微博视频号收益政策"，AI 给出 2023 年的旧政策。

根因是知识库更新不及时，RAG 检索到旧文档，LLM 未做时效判断。

**面试官**：怎么解决的？

**候选人**：三层解法——

**短期**：增加 "知识截止日期" 提示，对政策类问题加免责声明；**中期**：接入文档更新时间戳，优先返回最近 3 个月文档；**长期**：建立知识库自动同步机制，源文档更新后 24h 内同步到向量库。

最终 thumbs down 率从 40% 降到 12%，运营团队使用频次提升 2 倍。沉淀了 "时效敏感型知识" 的处理 SOP。

**面试官**：eufyMake 如果要做 AI 设计功能，你觉得最大的风险点可能是什么？

**候选人**：三个风险——

**生成质量不稳定**：用户用同样描述两次结果差异大，影响信任；**版权风险**：AI 生成模型与现有设计相似，引发法律问题；**打印失败**：AI 生成模型好看但打不出来，用户挫败感强。

应对是质量上引入种子锁定 + 版本控制；版权上训练数据清洗 + 相似度检测；打印上做可制造性检查。

## 第五题：协作冲突（28:00-35:00）

**面试官**：做 iOS 系统级合作时要协调多个团队，遇到过最大的冲突是什么？

**候选人**：最大的冲突是 Spotlight 索引频率的技术限制 vs 用户体验。

产品需求是 "准实时"，发博后 5 分钟内可搜到。但 iOS Spotlight API 有频率限制，强行突破可能导致 App 被系统降权。

**面试官**：怎么解决的？

**候选人**：数据说话 + 分层方案。

分析用户行为数据发现，80% 搜索发生在发博后 1 小时内。所以设计分层：热搜和大 V 内容走实时通道（5 分钟延迟），普通内容走批量索引（30 分钟延迟）。

A/B 测试验证：分层方案搜索成功率 98%，强行实时的方案因限流实际只有 85%。用数据说服了产品和研发团队。

**面试官**：如果研发团队说 "AI 生成 3D 模型技术上还做不到商用质量"，你会怎么推进？

**候选人**：先理解技术边界：是生成质量不行，还是后处理修复成本高？

然后找降级方案：AI 生成草图 → 人工精修 → 打印，而非一步到位。

分阶段落地：Phase 1 做 AI 辅助设计（参数建议、自动修复），Phase 2 做简单几何体生成，Phase 3 等复杂模型技术成熟。拿竞品案例（Meshy.ai）说明可行性。
