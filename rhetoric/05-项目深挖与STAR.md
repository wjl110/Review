# 项目深挖与 STAR 话术

面试前把下列每条准备 **1 分钟版 + 3 分钟版**，并核对简历上的数字是否仍为最新口径。

## 1. 微博内部 AI 智能体 / RAG 提效

**Situation**  
企业内部知识分散在 Wiki，运营与博主场景需要**可控幻觉、可检索、可工具调用**的智能体能力。

**Task**  
拆解业务场景，选型工具与 UI，搭建原型并推动迭代与上线反馈闭环。

**Action**  

- 需求与竞品对标（如 Google AI Studio、Gemini Enterprise 等思路）。  
- LangChain + Ghost / Big Model / Shell 等基础能力搭建原型；架构覆盖多模态解析、规划、决策、执行。  
- 博主文本切块、Embedding、向量库、检索控幻觉；function call 对接内部博主数据库。  
- 长短期记忆、CoT / ReAct / RAG、Multi-Agent 协作与工作流接口。  
- A/B、数据分析、员工反馈驱动迭代；社区/GitHub 侧动作（按实际说）。

**Result（以简历为准）**  
累计使用约 **1500+ 人次**；收集反馈约 **58 条** 驱动迭代。

**刁钻追问**

- **「RAG 效果不好你怎么办？」**  
  → 先分场景看 bad case：是**切块**、**embedding**、**重排**、还是**生成约束**问题；加评测集与人工抽检，小步实验（chunk 策略、hybrid search、query rewrite、拒答策略）。

- **「你怎么证明不是玩具项目？」**  
  → 用人次、留存/复访（若有）、业务侧采纳流程、以及反馈闭环次数说明**真实使用**。

## 2. iOS 系统级合作与功能落地

**Situation**  
iOS 26 端侧模型与 App Intents 等带来**系统级入口**机会，需要明确微博可落地的边界。

**Task**  
输出 PRD，定义 Spotlight schema、埋点与灰度上线策略，协同 UI/UE 与研发排期联调。

**Action**  
梳理端侧模型、意图识别与微博可结合方向；Shortcut / Spotlight / Widget / App Intents 等入口设计；布码 schema、数据分析策略；推动测试、灰度、上线。

**Result（以简历为准）**  
约 **11 月初** 上线 App Store；日均使用约 **30,962.8**（可持续优化叙事）。

**刁钻追问**

- **「苹果生态政策变了你怎么办？」**  
  → 核心能力抽象成「**意图与内容服务**」，入口层做适配；预留多入口与降级方案。

## 3. AIGC 视频创新实验室

**Situation**  
运营侧需要**低成本、可批量**的 AIGC 生产方案，同时跟踪海外工具与模型演进。

**Task**  
评估工具与模型质量，搭建私有化工作流，沉淀可复用 Sop。

**Action**  
Sora2、即梦、可灵、Nano Banana、GPT-4o 等评测；LoRA、ComfyUI、WebUI；数据到训练/推理/部署管线；Prompt 工程支撑博文-剧本-分镜-生图-视频链路。

**Result（以简历为准）**  
**20+** 篇技术方案/Sop；**30+** 工具挖掘；「围脖侠」全运会 AIGC 视频约 **50 万+** 浏览，互动 **500+**。

**刁钻追问**

- **「版权与合规你怎么控？」**  
  → 素材来源、可商用模型/API、水印与标识、内审流程（按微博实际流程说）。
