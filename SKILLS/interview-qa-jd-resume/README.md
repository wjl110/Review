# interview-qa-jd-resume

根据 **岗位 JD** 与 **个人简历**，生成 **面试问答全流程** Markdown（匹配度、自我介绍、项目深挖、场景题、行为面、反问、模拟节奏）。执行规范面向 Cursor Agent，说明文档面向人类。

## 快速开始

1. 在对话中 **@** 引用本目录下的 `SKILL.md`，或说明「按 interview-qa-jd-resume 流程执行」。
2. 提供 **JD**：本地路径（图片 / PDF / 文本）或招聘页 URL。  
3. 提供 **简历**：本地路径（PDF / 图片 / Markdown 等）。  
4. 指定 **输出路径**（可选）；默认在与 JD 同级或你指定的文件夹生成 `面试问答全流程-{公司}-{岗位简写}.md`。

## 目录说明

| 文件 | 说明 |
|------|------|
| [SKILL.md](./SKILL.md) | Agent 执行步骤、输出模板、答题原则（**主规范**） |
| [SOUL.md](./SOUL.md) | 元信息：用途、输入输出、`~/.cursor/skills` 挂载方式 |
| [设计方案.md](./设计方案.md) | 架构、数据流、质量门禁与扩展方式 |
| [对话示例.md](./对话示例.md) | 用户与 Agent 的示意对话与反例 |
| [README.md](./README.md) | 本文件：入口与索引 |

## 让 Cursor 自动发现 Skill（可选）

将本目录 **复制或软链接** 到：

`~/.cursor/skills/interview-qa-jd-resume/`

勿使用 `~/.cursor/skills-cursor/`（系统保留）。

## 维护约定

- 改流程 → 更新 `SKILL.md`。  
- 改调用方式或路径约定 → 同步 `SOUL.md` 与本文「快速开始」。  
- 改整体架构 → 更新 `设计方案.md`。

## 许可与免责

产出仅供个人面试准备参考；请确保 JD 与简历的使用符合招聘方与个人信息的合规要求。
