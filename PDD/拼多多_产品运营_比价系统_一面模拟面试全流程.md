## 一、一面面试官通常看什么

一面（业务初面）常见目标：**判断你是否能扛住「数据 + 策略 + 跨部门推进 + 算法协同」的复合型运营工作**，并快速筛掉「只会写文档/只会接需求」的候选人。


| 维度    | 面试官心里在问什么                | 你要交付的证据                  |
| ----- | ------------------------ | ------------------------ |
| 问题定义  | 你能不能从业务目标拆到可量化指标？        | 目标、指标、口径、优先级             |
| 数据与实验 | 你会不会用数据说话、做验证与复盘？        | A/B、漏斗、抽样、归因思路           |
| 算法协同  | 你能不能结构化描述 bad case、推动迭代？ | 标注规范、case 分层、验收标准        |
| 行业与策略 | 你对电商/定价/成本有没有基本商业常识？     | 平台规则、类目差异、毛利/补贴逻辑（可点到为止） |

| 稳定性   | 你会不会轻易甩锅、沟通是否职业化？        | 离职原因、冲突处理、复盘方式           |

## 二、岗位拆解（JD → 能力模型）

### 2.1 岗位职责翻译成「面试官语言」

1. **模型/效果优化（比价准确性）**
  - 你要能：**定义什么是“错”**（漏比、错比、时效差、覆盖不足、同品不同款、规格不一致等）  
  - 你要能：**把错例变成可训练/可规则化/可检索增强的输入**（数据、标注、特征、规则边界）  
  - 你要能：**和算法对齐验收**（离线指标 + 线上业务指标 + 抽样质检）
2. **策略优化（定价/比价策略体系）**
  - 你要能：把运营目标拆成阶段性策略（例如先解决覆盖率还是先解决头部类目准确率）  
  - 你要能：策略上线后的复盘机制（看哪些人群/类目/场景被误伤）
3. **行业研究（平台政策 & 成本结构）**
  - 你要能：说明你会如何建立信息源、如何结构化沉淀（政策变更、类目成本驱动因素）  
  - 你要能：把研究输出落到决策支持（不是写 PPT 自嗨）
4. **跨部门协同**
  - 你要能：讲清楚**谁依赖谁、什么时候对齐、如何降低返工**（产品/算法/数据/采销/风控等）

### 2.2 任职要求对照（你该怎么答）

- **2 年以上产品运营/产品经验**：你可用「微博管培 + AIGC 实习 + 研究/项目」拼出**高密度执行**，但要诚实界定年限，强调**结果与方法论可迁移**。  
- **数据分析 + 沟通 + 推进**：用 Kaggle 房价、A/B、埋点与灰度上线案例证明。  
- **加分：定价/成本/供应链/电商/AI**：你最强的是 **AI 侧协同 + 数据链路 + bad case 闭环**；电商定价直接经验不足时，用「同类问题结构」对齐。

## 五、模拟问答全流程（按一面常见顺序）

> 回答公式（可用 STAR，但别机械）：**背景 1 句 → 目标/指标 → 你做了什么决策 → 结果 → 复盘/下一步**。

### 5.1 开场与简历深挖

**Q1：你为什么投「比价系统/定价」？**  
**答**：我对“价格是否准确、是否可比、是否可解释”这类问题天然敏感；我过去在 AI 项目里大量处理 **匹配一致性、bad case、指标体系与迭代机制**，方法论上跟比价很像。希望把能力落到交易场景里，学习平台规则、供应链与成本结构，把策略做得更扎实。

**Q2：你对比价系统的理解是什么？**  
**答**：狭义是**同品识别 + 价格抓取/对齐 + 时效与覆盖**；对业务来说，比价结果会进入**定价、补贴、流量、风控**等策略链路。产品运营的关键通常是：定义错例类型、建立质检与抽样、推动算法/规则迭代，并用线上指标验证“更准”是否带来“更好的业务结果”。

**Q3：你没有电商核价经验，你怎么补齐？**  
**答**：我会用三步补齐：先快速建立**类目与 SKU 规格口径**（什么是同品、哪些字段必须一致）；再把**错价/漏比**按场景分层做清单；最后每周固定节奏和算法、业务方对齐**优先级与验收**。我过去在 bad case 与跨团队推进上做过完整闭环，缺的主要是行业知识，我会用文档化沉淀加速学习。

### 5.2 业务题（比价/定价向）

**Q4：如果你发现线上“比价准确率”下降，你怎么排查？**  
**答**：先确认是**数据链路问题**还是**模型/规则问题**：看抓取失败率、字段缺失、类目映射变更、对手页面改版；同时做**分层抽样质检**（头部类目/高 GMV SKU/新发品）。把问题拆成覆盖、匹配、时效、异常值四块，每一块给**可验证假设**与**短期止血方案**（规则/黑名单/人工复核）与**中期优化**（特征/训练数据/检索策略）。

**Q5：你怎么定义和收集 bad case？**  
**答**：先统一**错误标签**（非同品、规格不一致、价格非同款、活动价/券后价口径不一、时效滞后等）；再定**采样策略**（按类目、按 error rate、按投诉/工单）；输出给算法时要带**可复现输入**（链接、截图、结构化字段）和**期望输出**；最后设**回归集**避免修 A 坏 B。

**Q6：运营目标和模型指标冲突怎么办？**  
**答**：把冲突翻译成**约束条件**：例如业务要扩覆盖，模型要控误报，那就分阶段：**先保核心类目准确率，再扩长尾**，或用**双阈值策略**（高置信自动、低置信人工/二次校验）。关键是让业务方看到**取舍曲线**，用数据做共同决策。

**Q7：跨部门项目你怎么推进？（结合 iOS 项目）**  
**答**：先对齐**目标与里程碑**（上线窗口、灰度策略、核心指标），把依赖拆成**接口/埋点/实验/风控**清单；每周固定同步风险与变更；上线后用数据复盘而不是“感觉上线成功”。（准备 1 个你真实遇到的冲突与解决方式。）

### 5.3 数据题（一面常考“思路题”）

**Q8：你会关注哪些核心指标？**  
**答**：分层看：  

- **效果**：准确率/误报率/漏报率、TopN 类目表现、抽检一致性  
- **覆盖**：可比价 SKU 覆盖率、更新时效  
- **业务**：与策略相关的中间指标（例如错价导致的损失估计、活动价异常占比——口径需和业务确认）  
- **系统**：抓取成功率、解析失败、数据延迟

**Q9：A/B 测试你会怎么设计（类比你在微博的经验）**  
**答**：明确**主指标与护栏指标**；保证样本量与分流随机；避免外部大促/政策突变干扰；实验结束做**分层分析**（不同类目是否一致）；若提升不显著，输出假设：是样本不足、还是改动只影响部分人群。

### 5.4 行为题

**Q10：你最有成就感的事？**  
**答**：优先讲 **iOS 系统级合作上线 + 指标** 或 **RAG/Agent 从 0 到 1 + 使用量/反馈闭环**（二选一，讲透）。

**Q11：你失败或复盘最深刻的一次？**  
**答**：准备真实案例：问题是什么 → 你哪里判断错 → 怎么改流程/指标/协作 → 后来结果如何。（避免甩锅给同事。）

**Q12：你如何处理多任务并行？**  
**答**：按**业务影响/截止时间/依赖**排序；把任务拆到可交付粒度；对高风险项提前暴露依赖；用文档记录决策，减少口头扯皮。
