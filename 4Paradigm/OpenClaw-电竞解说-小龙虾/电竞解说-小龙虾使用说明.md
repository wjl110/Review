# 电竞解说 · 小龙虾（钳钳）使用说明

> **适用对象**：在 OpenClaw 中启用「小龙虾风格」电竞解说人设（昵称 **钳钳**）的运营与使用者。  
> **配套目录**：与本文件同级的 `SOUL.md`、`SKILL.md`、`设计方案.md`、`对话示例.md`、`README.md`。

---

## 1. 这是什么

**钳钳**是面向 **MOBA / FPS 等电竞场景** 的解说向智能体人设：热血、快节奏、幽默克制，负责 **解说词、战术白话、版本速读、新手引导** 等输出。  

完整人格、边界与话术规范见 `**SOUL.md`**；版本补丁类结构化输出流程见 `**SKILL.md**`；产品级设计与功能规划见 `**设计方案.md**`；可直接照抄或改写的对话样例见 `**对话示例.md**`。

## 3. 在 OpenClaw 里如何「加载」

OpenClaw 会在**每次新会话**把工作区根目录下的固定文件注入系统提示（Project Context）。本套件推荐两种落地方式（可同时使用）：

### 3.1 推荐：合并进工作区 `MEMORY.md`

将本目录下各 `.md` 的正文**按章节合并**到 Agent 工作区根目录的 `**MEMORY.md`**（文首注明来源目录与同步日期），并保证 `agents.defaults.bootstrapMaxChars` 足够大，避免注入截断。  

这样 **无需改根目录 `SOUL.md`**，也能在会话里读到钳钳的完整设定；若与根目录 `SOUL.md` 人设并存，请在 `MEMORY.md` 文首写明**优先级**（例如：电竞场景以钳钳为准）。

### 3.2 辅助：工作区内目录副本

将整个 `OpenClaw-电竞解说-小龙虾` 文件夹复制到工作区子目录（例如 `4Paradigm-import/OpenClaw-电竞解说-小龙虾/`），便于用 `**read` / `edit`** 单文件维护或与源目录 **diff**。  
**注意**：仅放在子目录**不会**自动注入上下文，仍需通过 `MEMORY.md` 或根目录标准引导文件参与会话注入。

### 3.3 可选：向量记忆 `extraPaths`

若已在 `openclaw.json` 中配置 `agents.defaults.memorySearch.extraPaths` 指向包含本套件的上一级目录，可通过 `**memory_search` / `memory_get`** 检索相关内容。  
这与「每轮系统提示注入」是不同机制；**开场就要人设稳定**，仍建议以 `MEMORY.md` 或根 `SOUL.md` 为主。

## 4. 日常使用：怎么对话

### 4.1 本机命令行（测试 / 脚本）

```bash
openclaw agent --agent main --message "你是钳钳。我想聊英雄联盟海克斯大乱斗最新版本的刁钻玩法，大乱斗没有「必胜公式」，但咱们可以拆解「怎么把随机性玩出花」，如何在最大随机性下增加最大可玩性和胜率～"
```

将下划线处换成实际游戏名；需要版本解读可提「按 SKILL 里的版本简报结构输出」。

### 4.2 已绑定渠道（飞书 / WhatsApp 等）

在对应渠道里直接发需求即可，例如：

- 「钳钳，用路人能听懂的话解释这波为什么要先动龙。」
- 「王者荣耀这次更新，按你的版本简报模板给我一版，我只贴了公告摘要：……」

### 4.3 写 Prompt 的小技巧

1. **先交代场景**：游戏名、模式（排位/匹配）、段位区间、单排还是组排。
2. **说清产出**：要解说词 / 教学步骤 / 版本 Meta 简报。
3. **给事实材料**：官方公告原文或赛果；不要要求模型编造未给出的数值改动。
4. 需要结构化输出时，可直接引用 `**SKILL.md`** 中的章节标题与模板。

## 5. 合规与边界（必读）

本套件与 `**SOUL.md**` 一致，核心包括：

- **不提供**：外挂、代练、租号买卖、博彩、开盒与人身攻击指引等。  
- **不保证**段位与胜率；强度类表述需区分 **事实** 与 **观点**，并加限定语。  
- 触及代练、租号、稳赢话术、长时间游戏、氪金等场景时，应出现 `**SOUL.md` 规定的风险提示块**（钳钳口吻）。

使用者与运营方需遵守平台规则与当地法律法规。

## 6. 维护与同步建议

1. **改稿流程**：在本仓库（或 `Aliyun/4Paradigm` 权威目录）修改 → 再同步到 OpenClaw 工作区 `MEMORY.md` 与/或 `4Paradigm-import` 副本。
2. **版本号**：重大人格或技能变更时，同步更新 `SOUL.md` / `SKILL.md` 文末版本说明与 `README.md`。
3. **Review**：部署前核对 `openclaw config validate`、网关已重启，并用 `openclaw agent ... --json` 查看 `systemPromptReport` 中 `MEMORY.md` 是否注入、是否 `truncated: false`。

更完整的 OpenClaw 集成步骤可参考工作区文档 `**OpenClaw-4Paradigm-集成最佳实践.md`**（若已拷贝到本机 `clawd` 目录）。

## 7. 常见问题


| 问题       | 处理方向                                                        |
| -------- | ----------------------------------------------------------- |
| 回复不像钳钳   | 确认 `MEMORY.md` 或根 `SOUL.md` 是否已注入；Prompt 中明确「按 SOUL.md 人设」。 |
| 版本解读胡编数值 | 只基于用户提供的官方文本；无公告则按 `SKILL.md` 输出「待核实」与检查清单。                 |
| 与人设冲突    | 根目录 `SOUL.md` 与 `MEMORY.md` 内嵌人设并存时，在 `MEMORY.md` 文首写清优先级。  |
| Token 很贵 | 缩短 `MEMORY.md` 中低频章节，或提高检索依赖、减少一次性注入体积（需权衡效果）。              |
