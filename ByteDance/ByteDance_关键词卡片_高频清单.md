# 字节面试关键词卡片

> 只记触发词，现场拼句子

---

## 卡片1：RAG项目（微博智能体）

```
背景：    客服重复咨询 → 人审压力大
目标：    降本增效 + 提升响应速度
动作：    知识库切块 + Embedding + 检索增强 + 幻觉检测
数据：    1500+ 人次使用、80% 准确率、3天上线
难点：    幻觉控制 → 引入置信度阈值 + 人工兜底
结果：    人审工作量下降 XX%、用户满意度提升
```

**触发词**：切块、Embedding、向量库、检索、幻觉、阈值、兜底、1500、80%、3天

---

## 卡片2：AIFlow / Workflow

```
背景：    多 Agent 协作需求
目标：    标准化流程 + 降低开发门槛
动作：    MCP 协议 + A2A 通信 + 可视化编排
数据：    3周 开发周期、1 项专利（申请中）
难点：    节点异常处理 → 重试机制 + 熔断策略
结果：    复用率提升、开发周期缩短
```

**触发词**：MCP、A2A、编排、节点、重试、熔断、3周、专利

---

## 卡片3：电商治理（JD匹配）

```
背景：    商家/内容违规风险
目标：    机审准召率提升 + 人审成本下降
动作：    规则库 + 知识库 + 案例库 + 大模型方案编排
数据：    机审覆盖率、准确率、召回率、人审介入率
难点：    新型违规识别 → 意图识别 + 多模态审核
结果：    治理效率提升、坏case闭环
```

**触发词**：规则库、知识库、案例库、准召率、意图识别、多模态、闭环

---

## 卡片4：Prompt工程

```
背景：    大模型输出不稳定
目标：    结构化 + 可复现
动作：    Prompt 2.0（角色+约束+示例+输出格式）
数据：    BadCase 收敛速度提升、迭代周期缩短
难点：    提示词注入攻击 → 输入过滤 + 输出校验
结果：    输出稳定性提升、维护成本下降
```

**触发词**：Prompt 2.0、结构化、角色、约束、BadCase、注入攻击、过滤

---

## 卡片5：数据分析

```
背景：    业务决策需要数据支撑
目标：    埋点完整 + 指标清晰
动作：    A/B 实验 + 漏斗分析 + 归因模型
数据：    转化率、留存率、覆盖率、准确率
工具：    SQL、Python、Tableau、神策
```

**触发词**：A/B、漏斗、归因、转化、留存、埋点、神策

## 卡片6：离职/转行（预备话术）

```
核心：    寻求更大平台 + 技术深耕 + 业务纵深
不说：    加班多、工资低、领导不行
可说：    希望聚焦大模型应用、看好电商赛道、字节技术氛围
迁移：    微博内容治理 → 电商内容治理（逻辑相通）
         RAG经验 → 知识库建设（能力复用）
```

**触发词**：聚焦、深耕、迁移、相通、复用、技术氛围

## 卡片7：反问环节（8个问题）

```
业务：    团队当前的大模型应用落地阶段？
         电商治理场景最棘手的case类型？
协作：    运营与算法/产品的协作模式？
         需求从提出到上线的典型周期？
成长：    新人前3个月的典型成长路径？
         有大模型运营方向的学习资源推荐？
氛围：    团队的技术分享频率和形式？
         对这个岗位的期望优先级排序？
```

**触发词**：落地阶段、协作模式、成长路径、优先级

## 使用说明

1. **面试前1小时**：快速过一遍所有卡片，唤醒记忆
2. **进场前5分钟**：只看「数字一览」和「应急锦囊」
3. **被问到了**：用卡片上的触发词组织语言，不用背整句
4. **卡住了**：看一眼对应卡片的「数据」行，用数字撑场面
