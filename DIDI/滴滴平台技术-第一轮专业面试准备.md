**Q1：你在微博做的 RAG 智能体，架构是怎样的？如何保证高可用和可扩展？**

**参考思路：**
- **架构**：多模态输入 → 解析与路由 → 规划/决策（ReAct、CoT）→ 执行（Function Call 调内部 API/数据库）→ 输出；RAG 侧：切块、Embedding、向量库检索、与 LLM 结合控制幻觉。
- **高可用/扩展**：检索与 LLM 解耦，检索失败可降级为纯模型回答；向量库和 API 可水平扩展；Agent 状态与记忆可持久化，便于重试与恢复。

**Q2：你提到多 Agent 协作，和网约车里的「多服务、多系统」协作有什么相似和不同？**

- **不同**：网约车更强调强一致性、时延 SLA、事务与补偿；Agent 协作更偏最终一致、语义层协调。可以接着说：自己希望把在 Agent 里学到的「清晰边界、可观测、可降级」用到分布式业务系统里。

**Q3：你做的更多是 AI/产品向，为什么投平台技术（后端/架构）？**

**参考思路：**

### 2. 计算机基础（按 JD：MySQL、Linux、网络、操作系统、分布式）

**Q4：MySQL 如何做高可用？主从、半同步、MHA/Orchestrator 了解吗？**

**参考要点：**
- 主从复制（异步/半同步）：从库拉 binlog 重放，半同步保证至少一个从库落盘再返回，减少主库宕机丢数。
- 高可用方案：MHA、Orchestrator 等做故障检测与主从切换；Paxos/Raft 类方案（如 MySQL MGR）做多节点一致性与自动选主。
- 可补充：读多写少时读写分离、从库延迟监控、主从数据一致性校验（如 pt-table-checksum）。

**Q5：MySQL 索引为什么用 B+ 树？和 B 树、哈希索引的区别？**

**参考要点：**
- B+ 树：多路平衡、高度低，适合磁盘；叶子存数据且有序链表，范围查询和排序友好；非叶子只存键，扇出大。
- B 树：非叶子也存数据，范围查询不如 B+ 树。
- 哈希：等值 O(1)，但不支持范围、排序，不适合做默认索引结构。

**Q6：分布式系统里，如何保证「高可用」？负载均衡、熔断、限流你了解多少？**

**参考要点：**
- **高可用**：无单点（多副本、多可用区）、故障转移（健康检查 + 自动摘除/切换）、容灾与演练。
- **负载均衡**：LVS/Nginx/云 LB；策略：轮询、加权、最少连接、一致性哈希（有状态时）。
- **熔断**：调用失败率/超时超过阈值时熔断，避免雪崩；半开探测恢复。
- **限流**：令牌桶、漏桶、滑动窗口；应用层或网关限流，保护下游。

**Q7：TCP 和 UDP 区别？TCP 如何保证可靠传输？**

**参考要点：**
- 区别：TCP 面向连接、可靠、有序、流控与拥塞控制；UDP 无连接、尽最大努力、低开销。
- 可靠机制：序列号与确认、超时重传、滑动窗口、拥塞控制（慢启动、拥塞避免、快重传/快恢复）。

**Q8：进程和线程的区别？多线程如何保证线程安全？**

**参考要点：**
- 进程：资源分配单位，有独立地址空间；线程：同一进程内的调度单位，共享地址空间，切换成本低。
- 线程安全：加锁（互斥锁、读写锁）、无锁结构（CAS）、线程局部存储、不可变对象；注意死锁（顺序、超时、死锁检测）。

### 3. 业务与场景（出行、分单、营销、高并发）

**Q9：网约车「分单」系统，如果让你设计，会考虑哪些核心模块和难点？**

**参考思路：**
- **核心模块**：订单接入与校验、司机/运力池、匹配与调度（距离、时间、运力状态、策略权重）、派单执行、状态机（订单/司机状态流转）、超时与改派、对账与计费。
- **难点**：高并发下的匹配性能与公平性、实时位置与ETA、策略可配置与 A/B、一致性（订单状态、库存）、降级与熔断（例如派单服务不可用时的降级策略）。

**Q10：如何设计一个能抗「高并发、海量请求」的接口？**

**参考要点：**
- 接入层：限流、防刷、网关聚合与路由。
- 服务层：无状态、水平扩展、异步与消息队列削峰、缓存（多级）、读写分离与分库分表思路。
- 存储与依赖：连接池、慢查询与索引优化、依赖熔断与超时。
- 可观测：监控、链路追踪、压测与容量规划。

**Q11：缓存和数据库一致性怎么保证？先更新库再删缓存、延迟双删等了解吗？**

**参考要点：**
- 常见做法：Cache Aside（读时先读缓存，miss 再读库并回填；写时先写库再删缓存，避免先更新缓存再写库的不一致窗口）。
- 延迟双删：写库后删缓存，再延迟一段时间再删一次，缓解读请求把旧数据写入缓存的并发窗口。
- 可补充：设置合理过期时间兜底；强一致场景用分布式锁或串行化，但会牺牲性能。

### 4. 编程语言（JD：Golang/Java/C++/PHP 至少一门）

**Q12：你最熟哪门语言？若用 Java，HashMap 原理、并发安全的 Map 有哪些？**

**参考要点（若答 Java）：**
- HashMap：数组 + 链表/红黑树；hash 计算、扩容、树化阈值。
- 并发安全：ConcurrentHashMap（分段锁/JDK8 后 CAS + synchronized 桶）、Hashtable、Collections.synchronizedMap。
- 若更熟 Python：可说明平时用 Python 做 AI/数据多，对 Java/Go 的并发容器有学习，愿意在岗位上深入。

**Q13：Golang 的 goroutine 和 channel 了解吗？和 Java 线程、线程池的对比？**

**参考要点：**
- Goroutine：轻量级用户态调度，M:N 映射到系统线程；channel 用于通信，避免共享内存。
- 对比：Go 更易写高并发 IO；Java 线程池显式控制并发度，生态成熟。网约车后端不少用 Go，可表达愿意在工作中深入。
