### A. 岗位与公司（3 题）

**Q1 你为什么投「企业智能知识平台」而不是偏 C 端或大模型基座？**

- **要点**：企业知识强调 **治理、复用、合规与成本**；你经历里 **内部 Wiki/RAG** 是同一类问题；希望做 **平台化** 而非单次项目。  
- **追问**：你认为车企/制造企业知识平台与互联网内容平台的**最大差异**是什么？  
- **避免**：空泛「看好蔚来」、只谈情怀不谈问题定义。

**Q2 JD 要求 5–7 年资深，你工作年限更短，你凭什么胜任「资深」？**  
- **意图**：压力面，看诚实度与可迁移能力。  
- **要点**：**先承认年限差距**；用 **高密度 ownership**（RAG 从 0–1 到规模化使用、iOS 全链路上线）+ **计算机背景**缩短「理解技术」成本；强调 **学习曲线与方法论**可迁移到企业场景。  
- **追问**：你如何用**指标**证明你不是「只做过功能」？  
- **避免**：硬吹等同 5 年经验、贬低 JD。

**Q3 如果让你用一句话定义「企业知识平台」，你怎么定义？**  
- **意图**：抽象能力。  
- **要点**：**可信、可发现、可复用、可度量**的知识服务层，支撑多 AI 应用；接入与治理是核心。  
- **追问**：「平台」和「项目制交付」差在哪？

## 4. 高光项目「深挖链」（各 1 条，从现象 → 实现 → 取舍）

### 链 ①：内部 RAG 智能体（最贴 JD）

1. **现象**：内部同学找博主/运营知识慢、口径不一。  
2. **目标**：可检索、可控幻觉、能接结构化数据。  
3. **实现**：切块、Embedding、向量库、检索策略、function call、工作流（CoT/ReAct 等按真实口径）。  
4. **闭环**：**1500+ 人次、~58 条反馈、A/B**。  
5. **取舍**：为何用该切块/该路由；Multi-Agent 的边界；不上图谱的原因（若属实）。

### 链 ②：iOS 系统级合作（体现「复杂协作 + 数据闭环」）

1. **现象**：抢占系统入口、提升打开路径。  
2. **实现**：PRD 能力边界、Spotlight schema、埋点、灰度。  
3. **结果**：**日均约 30962.8**（面试前核对表述）。  
4. **取舍**：系统能力不确定时的 **MVP** 与 **降级方案**。

### 链 ③：AIGC 实习—知识库 + 工作流（补「知识接入与运营侧」）

1. **现象**：内容生产要效率与风格一致。  
2. **实现**：知识库索引、BadCase、结构化 prompt、工作流、多模型比对、上线 AIFlow。  
3. **取舍**：**人工审核节点**放在哪一步；为何选某模型/某工具链。

## 8. 临场策略

### 8.1 电梯陈述

- **30 秒（主项目 = RAG 知识）**：  
  「我做企业内部 **Wiki/博主知识 RAG**，负责需求、检索与控幻觉、工具查库和 **A/B 闭环**；约 **1500+ 人次**使用、**几十条**反馈驱动迭代——这和贵司 **企业知识底座 + RAG** 高度同构。」

- **3 分钟（同一项目展开）**：  
  按 **用户与任务 → 架构分层（接入/索引/应用）→ 质量与幻觉 → 指标与实验 → 复盘与下一步平台化** 讲；中间插 1 个 **具体例子**（例如某类 query 的优化前后）。

### 8.2 5 条一面常见减分点与纠正

| 减分点 | 纠正方式 |
|--------|----------|
| 回避「5–7 年」差距 | **主动、简短承认**，转 **高密度成果与方法论**；问清职级是否可谈。 |
| 把 RAG 讲成「接大模型」 | 强调 **数据、检索、评测、权限/更新** 等产品工作。 |
| 编造图谱/推荐经历 | **诚实+补学路径**；用 RAG 检索经验做「相邻能力」。 |
| 只有定性没有指标 | 背熟 **人次、反馈条数、日均使用、A/B** 等可核对数字。 |
| 不懂企业场景约束 | 提前准备 **合规、权限、主数据、工单** 等关键词与「我会如何调研」。 |
